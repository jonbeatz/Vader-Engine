## 2. Technical Architecture & Backend Integration

The dashboard is a **Next.js 15 (App Router) + TypeScript + Tailwind CSS + shadcn/ui** web application that will be located at `ui/dashboard/`. In later phases it will be wrapped as a **Tauri 2 desktop app** (native binary ~10MB).

**Backend communication:**
- The dashboard will call your existing Node.js backend through API routes that execute `child_process` to invoke the canonical `msc:*` npm scripts.
- Do NOT rewrite the logic; simply wrap the existing CLI commands.

**Key npm scripts the dashboard must invoke (canonical list from package.json, 61‑point grader baseline):**
- `npm run start-project:gate` — full baseline: validate‑env → lint → grade → test:root
- `npm run msc:dev:example` — start minimal sandbox on port 3000
- `npm run msc:dev:payload` — start Payload CMS sandbox on port 3001
- `npm run msc:dev:tailwind` — start Tailwind/shadcn Path B sandbox on port 3002
- `npm run msc:kill-dev-port` — kill all ports (3000, 3001, 3002, 8080)
- `npm run msc:health -- --json` — return JSON diagnostics (ports, node version, env status)
- `npm run grade` — 61‑point integrity grader (non‑zero exit on failure)
- `npm run msc:lint` — Biome lint (0 errors)
- `npm run msc:test:root` — Vitest + audit
- `npm run msc:template -- list` — list all registered blueprints under `templates/`
- `npm run msc:template -- apply <template> --name="<name>" --target="<path>"` — scaffold a new project
- `npm run msc:template -- seed --template=<template> --target="<path>"` — seed mock data into a scaffolded project
- `npm run msc:validate-env` — environment secret leak check (must pass)
- `npm run verify:mcp` — MCP configuration validation

**Triple‑Sandbox Architecture (must be visually represented in the dashboard):**

| Sandbox | Stack | Port | Dev command | UI path |
|---------|-------|------|-------------|---------|
| **Minimal** | Next.js 15.5.7 + TypeScript + Vitest | 3000 | `npm run msc:dev:example` | Shield Path A |
| **Payload CMS** | Next.js 15.4.11 + Payload v3 + SQLite | 3001 | `npm run msc:dev:payload` | Shield Path A (admin) |
| **Tailwind/shadcn (Path B)** | Next.js 15.5.7 + Tailwind 3 + shadcn/ui | 3002 | `npm run msc:dev:tailwind` | Path B → `msc-shield.css` tokens |

**Health probes:** `npm run msc:health` monitors ports 3000, 3001, 3002, 8080.

**Session control (AI agent handoff — required):**
- `@Start-Project.md` — cold‑start initialization; runs `start-project:gate`, context ingestion, readiness check.
- `@End-Project.md` — session closeout; kills ports, git audit, log update, Handoff Block generation.

The dashboard must expose buttons for both session protocols.

## 3. Core Features (MVP)

### 3.1 Integrity Dashboard (Landing / Home)
- **61‑point grader overview:** Display “61/61 — 100% — Certified Stable” prominently. Provide a way to re‑run the grader and a detailed view of all 61 checks.
- Show the grade color‑coded: green if 61/61, yellow/red if anything else.
- Include a **Health Dashboard** section showing real‑time status of ports 3000, 3001, 3002, 8080 (listening / free). Use indicator dots.
- Display current Node version, CI baseline (Node 20.19.1), and package manager version (npm ≥10).
- **Action buttons:** “Run Full Gate” (`start-project:gate`), “Run Lint” (`msc:lint`), “Run Tests” (`msc:test:root`).

### 3.2 Sandbox Launcher
- Visual cards for the three sandboxes: Minimal, Payload CMS, Tailwind/shadcn (Path B).
- Each card shows: name, stack, port number, and current status (running/stopped).
- Buttons: **Start** (runs the respective `msc:dev:*` command), **Stop** (kills the port), **Open** (opens `http://localhost:<port>` in browser).
- Real‑time indication of which sandbox is active; logs pane shows terminal output.

### 3.3 Template Gallery & Project Scaffolding
- Fetch the list of available blueprints from `npm run msc:template -- list`.
- Display cards or a list of templates (portfolio, divi‑bridge, task‑manager, vader‑site).
- **“Create New Project” wizard:**
  - Select template from gallery.
  - Input fields: project name, target directory (with folder picker — native or fallback), toggle for including mock data (seed).
  - Submit → runs `msc:template apply` and optionally `msc:template seed`.
  - Show real‑time terminal output from the scaffolding process.
  - On success, display the path to the newly created project and optionally offer to open it in the system explorer.

### 3.4 Script Runner / Quick Actions
- A panel of frequently used scripts as one‑click buttons:
  - **Grade** (`grade`)
  - **Lint & Fix** (`msc:lint:fix`)
  - **Validate Environment** (`msc:validate-env`)
  - **Kill All Ports** (`msc:kill-dev-port`)
  - **Start Session** (runs `@Start-Project.md` protocol: gate + context)
  - **End Session** (runs `@End-Project.md` protocol: kill ports + log + git audit + Handoff Block)
- Each button click streams terminal output to a real‑time logs pane.

### 3.5 Environment Manager
- Display the status of `.env.local` (valid / missing placeholders) based on `msc:validate-env`.
- List required environment variables from `.env.example` (YOUR_* placeholders).
- Provide a safe interface to create/edit `.env.local` values (no terminal).
- Button to run `msc:validate-env` and show the result.

### 3.6 Real‑time Logs Viewer
- A pane that captures stdout/stderr from any command execution.
- Should support auto‑scroll and clearing.
- Use terminal aesthetic: monospace font, neon green for success, red for errors.

### 3.7 CI/CD & GitHub Integration
- Integration with your `release.yml` workflow.
- Option to trigger a release tag (v2.x.x) directly from the dashboard (calls `git tag` and pushes).
- Show latest release information (from GitHub API).

### 3.8 Recent Projects
- Keep a local history (localStorage or simple JSON file) of projects created via the scaffold wizard.
- Display them as a list with path, date created, and a button to open the folder / start dev server.

### 3.9 Operator Controls & Protocol Automation
- **Cold‑Start button:** Equivalent to `@Start-Project.md` — runs `start-project:gate`, performs context ingestion, and displays “System Ready”.
- **Session Close button:** Equivalent to `@End-Project.md` — kills ports, git status, logs update, prints Handoff Block.
