The Boilerplate is a Cursor-native full-stack development workspace.
It has a self-grading integrity system and three isolated sandbox
environments. The dashboard calls these existing npm scripts via
Next.js API routes using child_process — no new backend logic.

THE SANDBOXES (dev environments the user starts/stops):
  Sandbox A — "Minimal Frontend"
    Port: 3000
    Stack: Next.js 15.5.7 + TypeScript + Vitest
    Command: npm run msc:dev:example
    Status: shows running/stopped, uptime, port occupancy

  Sandbox B — "Full-Stack CMS"
    Port: 3001
    Stack: Next.js 15.4.11 + Payload CMS v3 + SQLite
    Command: npm run msc:dev:payload
    Status: shows running/stopped, CMS ready indicator

  Sandbox C — "Tailwind / shadcn"
    Port: 3002
    Stack: Next.js 15.5.7 + Tailwind 3 + shadcn components
    Command: npm run msc:dev:tailwind
    Status: shows running/stopped

  Dashboard itself:
    Port: 3010
    This is the dashboard — it shows its own port in the registry

THE GRADER (self-integrity system):
  Command: npm run grade
  Output: 61-point structural integrity check
  Result format: "61/61 (100%)" = PASS — shown in bright green
  Each check is labeled [PASS] or [FAIL] with a description
  CI requires 100% — failure blocks deployment
  This is the crown jewel feature of the project

KEY npm SCRIPTS (these all become buttons in the UI):

  LIFECYCLE:
    npm run msc:onboard        → Interactive first-time setup wizard
    npm run bootstrap          → Full workspace bootstrap + port cleanup
    npm run msc:quickstart     → Asset copies, credential checks
    npm run msc:validate-env   → Secret leak + placeholder audit
    npm run msc:kill-dev-port  → Purge ports 3000, 3001, 8080

  DEVELOPMENT:
    npm run msc:dev:example    → Start Sandbox A (port 3000)
    npm run msc:dev:payload    → Start Sandbox B (port 3001)
    npm run msc:dev:tailwind   → Start Sandbox C (port 3002)
    npm run msc:health         → System diagnostics
    npm run msc:health -- --json → Machine-readable health output
    npm run dev:recover        → Recover stuck dev server

  QUALITY:
    npm run grade              → 61-point integrity grader
    npm run msc:lint           → Biome lint + format check
    npm run msc:lint:fix       → Biome auto-fix
    npm run msc:test:root      → npm audit + root Vitest
    npm run msc:test:all       → All Vitest tests
    npm run verify:mcp         → MCP config validation
    npm run msc:shield:audit   → CSS namespace compliance check

  TEMPLATES / SCAFFOLDING:
    npm run msc:template -- list   → List available blueprints
    npm run msc:template -- apply  → Scaffold blueprint to new project
    npm run msc:template -- seed   → Write mock data payload
    npm run msc:new:component      → Generate Vader-compliant component

  MAINTENANCE:
    npm run msc:e2e            → Playwright E2E tests (ports 3000+3001)
    npm run msc:update         → Bump package versions across workspaces
    npm run clean:next         → Clear Next.js build cache
    npm run repair:sqlite      → SQLite repair utilities

AVAILABLE TEMPLATES (shown in template gallery):
